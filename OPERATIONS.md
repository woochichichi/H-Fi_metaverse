# ITO 메타버스 운영 가이드

관리자(admin)를 위한 운영 매뉴얼입니다.

## 1. 초대 코드 관리

### 생성 방법
1. PC: TopBar 우측 **⚙ 설정** 아이콘 클릭 → 관리자 패널
2. 모바일: 더보기 탭 → **관리자 패널**
3. "초대코드 관리" 섹션에서 **새 코드 생성**
4. 설정 항목:
   - **팀**: 증권ITO / 생명ITO / 손보ITO / 한금서 (선택)
   - **역할**: leader / member
   - **최대 사용 횟수**: 기본 10회
   - **만료일**: 선택 사항
5. 생성된 코드(FITO-XXXX-XXXX)를 팀원에게 공유

### 비활성화
- 코드 목록에서 토글 스위치로 활성/비활성 전환
- 비활성화된 코드로는 가입 불가

## 2. 사용자 관리

### 역할 변경
1. 관리자 패널 → "사용자 관리"
2. 사용자 목록에서 역할 드롭다운으로 변경
   - **admin**: 전체 관리 권한 (초대코드, 사용자, VOC 처리, 공지 작성, 평가)
   - **leader**: 팀 리더 권한 (VOC 처리, 아이디어 상태 변경, 공지 작성)
   - **member**: 일반 팀원 (VOC 작성, 아이디어 등록/투표, 쪽지)

## 3. VOC 처리 프로세스

### 상태 흐름
```
접수 → 검토중 → 처리중 → 완료
                        → 보류
```

### 처리 절차
1. VOC 센터에서 접수된 건 확인
2. VOC 카드 클릭 → 상세 보기
3. 상태 변경: 접수 → 검토중 → 처리중 → 완료
4. 양방향 익명 대화로 작성자에게 진행 상황 전달
5. 해결 시 "완료" 상태로 변경 + 해결 내용 기록

### 익명 원칙
- 익명 VOC의 작성자 ID는 저장되지 않음 (관리자도 확인 불가)
- 양방향 대화는 "작성자 / 관리자" 역할로만 구분

## 4. 공지사항 작성

### 시급성 3단계
| 시급성 | 의미 | 알림 방식 |
|--------|------|-----------|
| 🔴 **긴급** | 즉시 확인 필요 | 수집함 + 홈 배너 |
| 🟡 **할일** | 기한 내 확인 | 수집함 |
| 🔵 **참고** | 참고 사항 | 수집함 (낮은 우선순위) |

### 작성 절차
1. 공지 패널 → **+** 버튼
2. 제목, 내용, 시급성, 카테고리(일반/이벤트/활동보고) 선택
3. 파일 첨부 (최대 10MB, 이미지/PDF)
4. 고정 여부 선택 (고정 시 목록 상단 유지)
5. 등록 → 팀원들에게 자동 알림

### 읽음 추적
- 각 공지의 읽음 상태가 자동 기록됨
- 관리자는 누가 읽었는지 확인 가능

## 5. 평가 대시보드 활용

### 자동 기록 항목
| 활동 | 포인트 |
|------|--------|
| VOC 작성 | +3pt |
| 아이디어 등록 | +3pt |
| 아이디어 투표 | +1pt |
| 공지 읽음 | +0.5pt |
| 이벤트 참여 | +5pt |
| 쪽지 발송 | +1pt |
| 인적교류 참여 | +5pt |

### 대시보드 기능
- **팀별 히트맵**: 팀 간 활동 비교
- **개인별 요약**: 활동 카드 + 포인트 합계
- **CSV 내보내기**: 전체 데이터 다운로드

## 6. 일반 운영 팁

- **주 1회**: 미처리 VOC 확인 + 상태 업데이트
- **월 1회**: KPI 실적 입력 독려 + 평가 대시보드 점검
- **분기 1회**: 비활성 초대코드 정리, 사용자 역할 점검
- 긴급 공지는 꼭 🔴 시급성 설정 (홈 배너 표시)
- 익명 VOC/쪽지 답변 시 정중한 어조 유지
