# 한울타리 v2 — 대시보드 UI 설계 문서

> 브랜치: `v2` · 테마: **Warm Minimal (크림 + 코랄)**
> 원칙: 기존 DB/RLS/인증 체계는 **그대로 유지**, UI만 대시보드 형태로 재구성

---

## 1. 설계 원칙

1. **DB·RLS 변경 없음** — 기존 34개 마이그레이션은 그대로. 추가 필요 시 35+ 로 append-only.
2. **권한 체계 불변** — `admin / director / leader / member` 4단계, leader 팀 격리, 익명성 보호 모두 유지.
3. **기존 메타버스와 역할 취급 동일**
   - `admin`과 `director`는 코드 전역에서 **`isAdmin` 플래그 하나로 동일 취급** (`role === 'admin' \|\| role === 'director'`).
   - 둘의 유일한 차이: **`admin`만** 접근 가능한 메뉴 2개 = **초대 코드 / 차단 로그** (기존 `AdminPanel.tsx` 기준).
   - `leader`는 기존처럼 관리 메뉴(사용자 관리 / 평가 항목 / 평가 대시보드) 노출, **자기 팀만** 필터.
4. **v1 공존** — 사용자가 상단바 "디자인" 버튼으로 언제든 `classic(v1 메타버스) ↔ v2(대시보드)` 전환 가능. 선택은 `localStorage`에 저장.
5. **모바일은 기존 그대로** — `MobileLayout` 유지. v2는 PC(≥1024px)만 타겟.
6. **기존 React 컴포넌트 최대 재사용** — `VocCard`, `NoticeList`, `KpiChart`, `InboxBadge` 등. 신규 작성은 레이아웃/대시보드 위젯만.

**핵심 규칙 (기존 메타버스와 동일)**
- `admin` ≡ `director` (동일 `isAdmin` 플래그). 코드베이스 어디든 같이 처리.
  - **유일한 차이**: 초대 코드 / 차단 로그는 `admin`만 (`AdminPanel.tsx` roles 배열 기준)
- `leader`는 관리 섹션을 볼 수 있지만 모든 데이터가 **자기 팀(`profile.team`)만** 필터. UI + 기존 RLS 이중 방어.
- `member`는 관리 섹션 자체가 사이드바에 노출되지 않음.

## 4. 데이터 / 익명성 안전 가이드

### 익명 처리 체크리스트
- [ ] 모든 VOC 카드/리스트 렌더는 기존 `VocCard`/`VocList` 컴포넌트 재사용
- [ ] 익명 VOC는 작성자 아바타/이름 대신 "익명" 라벨만
- [ ] 익명 쪽지는 `sender_id=null`이면 "익명" 표시, admin UI에서도 동일
- [ ] 고민방(worry) 렌더 시 `author_id=null` 여부로 익명 배지 표시
- [ ] 모든 사용자 이름 렌더 지점에 `getDisplayName(profile, isAdmin)` 헬퍼 사용 (`isAdmin = role === 'admin' || role === 'director'`)

### RLS에 의존하되, UI 차원에서도 한 번 더 막기
| 검사 | 서버(RLS) | 클라이언트(UI) |
|---|:-:|:-:|
| leader가 타팀 VOC 상태 변경 | ✅ | ✅ 버튼 비노출 |
| member가 공지 작성 | ✅ | ✅ 메뉴 비노출 |
| member가 다른 사람 실명 조회 | ✅ `getDisplayName` | ✅ 동일 |
| 퇴사자 목록 노출 | ✅ `status='퇴사'` 필터 | ✅ 렌더 시 제외 |
| 익명 작성자 추론 | ✅ null 저장 | ✅ "익명" 라벨 강제 |

## 5. 사용자 플로우

### 최초 v2 진입
1. 사용자가 상단바 **"디자인"** 버튼 클릭 → `/ui-version` 진입
2. 4개 카드 중 Warm Minimal 선택 → `themeStore.version = 'warm'` 저장
3. `/` 리다이렉트 → **대시보드** (기본 랜딩)
   - 단, 미확인 **긴급 공지**가 있으면 공지 랜딩 페이지 먼저 (§5.2)

### 5.2 로그인 시 공지 랜딩 판별 규칙
1. 로그인 → `MainPage` 진입
2. `themeStore.version` 체크
   - `classic` → 기존 메타버스 맵
   - `warm` 외 3종 → 해당 대시보드 버전
3. **v2 대시보드**일 때 초기 라우팅:
   ```ts
   // 의사 코드 — 실제는 훅으로 구현
   const { data: urgentUnread } = useQuery(() =>
     supabase.from('notices')
       .select('id', { count: 'exact' })
       .eq('urgency', '긴급')
       .not('id', 'in', /* notice_reads where user_id = me */)
   );
   if (urgentUnread.count > 0) navigate('/notice-landing');
   else                         navigate('/dashboard');
   ```
4. 공지 랜딩에서는:
   - **긴급 공지**: 모두 "확인 완료" 버튼 클릭 → `notice_reads` insert → 하나라도 남으면 계속 표시
   - **할일/참고**: 같은 페이지에 보여주지만 **확인 의무 없음** (선택)
   - 모든 긴급을 확인 완료 → 자동 대시보드 리다이렉트 + 다음 로그인부터 공지 랜딩 건너뜀

### 디자인 재변경
- 상단바 "디자인" 버튼 언제든 활성 → `/ui-version`
